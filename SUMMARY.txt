================================================================================
TURING LLM BENCHMARK - COMPLETE SUMMARY
================================================================================

WHAT IT IS:
  A generic, external, invariant benchmark for LLM serving systems
  ├─ Validates correctness (4-layer validity gate)
  ├─ Measures performance under load (concurrent workload)
  ├─ Analyzes capacity (optional sweep)
  └─ Compares against immutable baseline (never temporal noise)

HOW IT WORKS:
  Three automatic phases when you run:
  python -m turing_bench run --endpoint http://localhost:8000 --phase baseline --stack-id mystack

  PHASE 1: SEQUENTIAL EXECUTION (Correctness)
    ├─ Runs 50 times, one request at a time
    ├─ Validates each output (sanity → structural → semantic → exact-match)
    ├─ STOPS if any validation fails
    └─ Collects: TTFT, latency, validity status, raw outputs

  PHASE 2: CONCURRENT WORKLOAD (Performance)
    ├─ Runs 500 requests at fixed RPS (e.g., 16 req/sec)
    ├─ Only runs if Phase 1 passed
    ├─ No validation (already done)
    └─ Collects: P50/P95/P99, CV, drift, spikes, distribution

  PHASE 3: SWEEP (Capacity - Optional, --sweep flag)
    ├─ Increases concurrency from 1 to 64
    ├─ 50 requests per level
    └─ Finds saturation point

================================================================================
METRICS COMPUTED
================================================================================

LATENCY METRICS:
  ├─ P50, P95, P99 (percentiles)
  ├─ Mean latency
  └─ CV% (coefficient of variation)

TTFT METRICS (Time-to-First-Token, streaming only):
  ├─ Mean, P50, P95
  └─ Only for streaming responses

STABILITY METRICS:
  ├─ Drift % (first half vs second half)
  ├─ Spike count (outliers > median × 2.5)
  ├─ Spike % (as percentage of total)
  ├─ P99/P95 ratio (fat tail indicator)
  └─ Distribution shape (right-skewed?)

================================================================================
CONTROL FLOW (YOUR VARIANCE DIAGRAM)
================================================================================

After Sequential Phase:
  ├─ Compute CV (coefficient of variation)
  │
  ├─ If CV ≤ 5%
  │  └─ GREEN: "Results reliable" → Continue
  │
  ├─ If 5% < CV ≤ 10%
  │  └─ YELLOW: "Results noisy" → Continue (warn)
  │
  └─ If CV > 10%
     └─ RED: "Results unreliable" → STOP (or warn)
        └─ Recommend: Stabilize system

After Concurrent Phase:
  ├─ Check stability metrics
  │  ├─ Drift > 5%? → Thermal throttling
  │  ├─ Spikes > 5%? → Background processes
  │  └─ P99/P95 > 1.5? → Fat tail (occasional slow)
  │
  └─ Report metrics
     ├─ Show performance improvement/regression
     ├─ Show validity gate result
     └─ Recommend actions if noisy
